# 校园网登录测试功能说明

## 功能概述
在 NetworkMonitor 应用中添加了一个"测试登录"按钮，可以直接从应用程序内发送校园网认证请求到 `2.2.2.2/quickauth.do`。

## 添加的功能

### 1. 新增"测试登录"按钮
- **位置**: 主界面右侧，"测试访问"按钮旁边
- **功能**: 点击后发送校园网登录请求

### 2. 登录参数配置
当前使用的登录凭证（在 `MainForm.cs` 的 `LoginTestButton_Click` 方法中）:

```csharp
var loginData = new Dictionary<string, string>
{
    { "userid", "23325024026" },      // 用户名
    { "passwd", "17881936070" },      // 密码
    { "wlanuserip", "" },
    { "wlanacname", "" },
    { "wlanacIp", "" },
    // ... 其他参数
};
```

### 3. 认证流程
1. 点击“测试登录”按钮
2. **第一步**: 访问门户页面 `http://2.2.2.2` 获取网络参数
   - 解析 HTML 中的重定向 URL
   - 提取 `wlanuserip`、`wlanacname`、`mac`、`vlan` 等参数
3. **第二步**: 构建包含全部参数的认证请求
4. 发送 GET 请求到 `http://2.2.2.2/quickauth.do`
5. 解析 JSON 响应
6. 显示登录结果

### 4. 响应处理
- **成功** (code = "0"): 显示成功消息框
- **失败** (code != "0"): 显示错误代码和错误信息
- **异常**: 显示网络错误或解析错误

## 使用说明

### 修改登录凭证
如需修改用户名和密码，请编辑 `MainForm.cs` 文件中的 `LoginTestButton_Click` 方法:

```csharp
{ "userid", "你的用户名" },
{ "passwd", "你的密码" },
```

### 查看日志
所有登录尝试都会记录到应用程序的日志窗口中，包括:
- 请求发送时间
- 用户名
- 响应状态码
- 返回代码和消息

## 实现细节

### 基于 portalUtil.js 的实现
该功能基于校园网认证页面的 `portalUtil.js` 文件中的 `checkSubmit()` 函数实现:
- 使用相同的请求参数结构
- 使用 GET 方法（而非 POST）
- 解析相同的 JSON 响应格式

### 两步认证机制
1. **第一步 - 参数获取**: 
   - 访问 `2.2.2.2` 门户页面
   - 服务器返回包含 `location.replace()` 的 HTML
   - 从重定向 URL 中提取用户的实际网络参数
   - 示例: `wlanuserip=192.168.199.76`, `mac=00:e2:69:5b:9c:ad`, 等

2. **第二步 - 认证请求**:
   - 将获取的参数与用户名密码组合
   - 发送到 `/quickauth.do` 接口
   - 服务器返回 JSON 响应表示认证结果

### 代码位置
- **主要代码**: `MainForm.cs` 第 355-523 行
- **主方法**: `LoginTestButton_Click` - 处理整个认证流程
- **辅助方法**: `ExtractParametersFromUrl` - 从 URL 提取查询参数

## 注意事项

1. **安全性**: 当前版本将密码硬编码在源代码中，不适合生产环境
2. **参数配置**: 部分网络参数（如 `wlanacname`、`wlanacIp` 等）当前为空，可能需要根据实际网络环境填写
3. **Cookie**: 原 JavaScript 实现使用 Cookie，当前 C# 版本未包含 Cookie 处理

## 改进建议

1. **配置文件**: 将用户名和密码移到配置文件或设置界面
2. **加密存储**: 使用加密方式存储凭证
3. **自动获取参数**: 自动检测并填充网络参数（wlanacname、mac 等）
4. **Cookie 支持**: 添加 Cookie 容器以支持完整的认证流程
